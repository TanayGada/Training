# Open Closed Principle

## Definition
- Software entities (classes, modules, functions, etc.) should be open for extension, but closed for modification.

## What does it Mean?
- We add new functionality without changing existing code.
- We use abstraction and polymorphism.


## Benefits of OCP
1.Extensibility: Add features with minimal effort.
2. Stability: Existing tested code stays untouched -> fewer bugs.
3. Maintainability: Modular code that’s easier to understand and change.
4. Scalability: New types or behaviors are easily plugged in.


## OCP Violation
```java
class PaymentProcessor {
    public void processPayment(String type) {
        if (type.equals("creditcard")) {
            System.out.println("Processing credit card");
        } else if (type.equals("paypal")) {
            System.out.println("Processing PayPal");
        }
    }
}
```
- Issue: We need to modify processPayment() every time a new payment method is added.

## OCP Compliant Design
#### Step 1: Create an Interface
```java
interface PaymentMethod {
    void pay();
}
```
#### Step 2: Implement Each Payment Type
```java
class CreditCardPayment implements PaymentMethod {
    public void pay() {
        System.out.println("Processing credit card");
    }
}

class PayPalPayment implements PaymentMethod {
    public void pay() {
        System.out.println("Processing PayPal");
    }
}
```

#### Step 3: Use Abstraction in the PayementProcessor
```java
class PaymentProcessor {
    public void processPayment(PaymentMethod method) {
        method.pay();
    }
}
```

#### Step 4: Add New Type Without Modifying Anything
```java
class UpiPayment implements PaymentMethod {
    public void pay() {
        System.out.println("Processing UPI");
    }
}
```

#### We use main function as
```java
public class Main{
    public static void main(String[] args){
        PaymentProcessor processor= new PaymentProcessor();

        PaymentMethod credit = new CreditCardPayment();
        PaymentMethod paypal = new PayPalPayment();

        processor.processPayment(credit); //Processing credit card payment
        processor.processPayment(paypal); //Processing PayPal payment
    }
}
```


## Key Points
1. Use Interfaces and abstraction
2. Rely on polymorphism to allow new behavior.
3. Avoid if-else or switch logic to handle different cases.
4. Apply OCP with Strategy, Factory, or Command design patterns.
5. Keep base classes or interfaces stable.
6. Favor composition over inheritance for greater flexibility.
