# Liskov Substitution Principle (LSP)

## Definition
- You should be able to replace an object of a superclass with an object of a subclass without breaking the application.

## What does it mean?
- A subclass shouldn’t override the behavior of the base class in a way that breaks the expected behavior.
- Subclasses must adhere to the contract established by the superclass.
- The “is-a” relationship should be meaningful in behavior, not just structure.
- Always ensure that a subclass truly behaves like the parent class it extends. If not, rethink the inheritance.

## Example 1: Color Classes
### Violation of LSP
```java
public class Green {
   public void getColor() {
      System.out.println("Green");
   }
}

public class Blue extends Green {
   public void getColor() {
      System.out.println("Blue");
   }
}

public class Main {
   public static void main(String[] args) {
      Green green = new Blue();  // Looks like Green, but prints Blue
      green.getColor(); // Output: Blue (unexpected)
   }
}
```
- we are expecting a “Green” object to behave like Green, but it behaves like Blue. This breaks LSP.
### Corrected Way (using Interfaces)
```java
public interface Color {
   void getColor();
}

public class Green implements Color {
   public void getColor() {
      System.out.println("Green");
   }
}

public class Blue implements Color {
   public void getColor() {
      System.out.println("Blue");
   }
}

public class Main {
   public static void main(String[] args) {
      Color color = new Blue();  // Now behavior is expected
      color.getColor(); // Output: Blue
   }
}
```
- Using interfaces, each class is clear about its own behavior, without misleading inheritance.


## Example 2: Bird/Pengium Problem
### Violation of LSP
```java
public interface Bird {
    void fly();
    void walk();
}

public class Penguin implements Bird {
    public void fly() { /* Penguin can't fly! */ }
    public void walk() { /* walks */ }
}
```
- Penguins cannot fly, but we are forced to implement fly() which violats LSP because not all Birds can fly.

### Better Design
```java
public interface Bird {}

public interface FlyingBird extends Bird {
    void fly();
}

public interface WalkingBird extends Bird {
    void walk();
}

public class Parrot implements FlyingBird, WalkingBird {
    public void fly() { /* Parrot flies */ }
    public void walk() { /* Parrot walks */ }
}

public class Penguin implements WalkingBird {
    public void walk() { /* Penguin walks */ }
}
```
- Now, each bird implements only the behaviors it actually supports.
