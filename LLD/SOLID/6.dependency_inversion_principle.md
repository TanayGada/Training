# Dependency Inversion Principle

## Definition
- High level modules should not depend on low level modules; both should depend on abstractions. 
- Abstractions should not depend on Implementation.  Implementation should depend upon abstractions

## What does it mean
- High level modules or low level modules should not depend on the actual implementation.
- They should depend on abstractions.
- We implement abstractions using interfaces

## More Detail
- Usually high level modules depend on low level modules.
- A change in low level module affects the working of the high level modules
- Also the unit testing of these high level module requires mocking of the low level module
- Solution, we make both low level and high level modules on the common interface

## Problem Without DIP
- Imagine a system where a LightSwitch class directly depends on a LightBulb class:
```java
// Low-level class
public class LightBulb {
    public void turnOn() {
        System.out.println("LightBulb turned on");
    }

    public void turnOff() {
        System.out.println("LightBulb turned off");
    }
}

// High-level class
public class LightSwitch {
    private LightBulb bulb;

    public LightSwitch(LightBulb bulb) {
        this.bulb = bulb;
    }

    public void operate(String command) {
        if ("ON".equals(command)) bulb.turnOn();
        else if ("OFF".equals(command)) bulb.turnOff();
    }
}
```

#### Problem:
- LightSwitch directly depends on LightBulb.
- We can’t easily switch to another device like Fan or TubeLight.


## Applying DIP
1. Introduce an abstraction (interface):
```java
// Abstraction
public interface SwitchableDevice {
    void turnOn();
    void turnOff();
}
```

2. Update LightBulb to implement the abstraction:
```java
public class LightBulb implements SwitchableDevice {
    public void turnOn() {
        System.out.println("LightBulb turned on");
    }

    public void turnOff() {
        System.out.println("LightBulb turned off");
    }
}
```

3. Now, modify the high-level module to depend on the abstraction:
```java
public class LightSwitch {
    private SwitchableDevice device;

    public LightSwitch(SwitchableDevice device) {
        this.device = device;
    }

    public void operate(String command) {
        if ("ON".equals(command)) device.turnOn();
        else if ("OFF".equals(command)) device.turnOff();
    }
}
```
4. Easily add/swap new Device
```java
public class Fan implements SwitchableDevice {
    public void turnOn() {
        System.out.println("Fan turned on");
    }

    public void turnOff() {
        System.out.println("Fan turned off");
    }
}
```
### In Main
```java
public class Main {
    public static void main(String[] args) {
        SwitchableDevice bulb = new LightBulb();
        LightSwitch bulbSwitch = new LightSwitch(bulb);
        bulbSwitch.operate("ON");
        bulbSwitch.operate("OFF");

        SwitchableDevice fan = new Fan();
        LightSwitch fanSwitch = new LightSwitch(fan);
        fanSwitch.operate("ON");
        fanSwitch.operate("OFF");
    }
}
```

## Benefits of DIP
1. Decouples high-level and low-level modules.
2. Enables easy swapping of components.
3. Increases testability (can use mocks/stubs).
4. Improves flexibility and maintainability.
