Aggregation is a concept in object-oriented programming where a class is made up of one or more objects from other classes. It represents a "has-a" relationship, indicating that one class contains or is composed of other classes. Aggregation is a type of association with a stronger relationship, but it does not imply ownership (as opposed to composition, which does).

### Real-Life Example: Car and Engine

Consider a real-life scenario where a car is made up of an engine, wheels, and other components. The car "has-a" engine, "has-a" set of wheels, etc. The engine can exist independently of the car, making this a suitable example of aggregation.

#### Class: Engine

```cpp
#include <iostream>
#include <string>

class Engine {
private:
    std::string type;

public:
    Engine(std::string t) : type(t) {}

    void start() {
        std::cout << "Engine of type " << type << " is starting." << std::endl;
    }

    std::string getType() const {
        return type;
    }
};
```

#### Class: Wheel

```cpp
class Wheel {
private:
    int size;

public:
    Wheel(int s) : size(s) {}

    int getSize() const {
        return size;
    }
};
```

#### Class: Car

```cpp
class Car {
private:
    std::string model;
    Engine* engine; // Aggregation: Car has an Engine
    Wheel* wheels[4]; // Aggregation: Car has Wheels

public:
    Car(std::string m, Engine* e, Wheel* w[4]) : model(m), engine(e) {
        for (int i = 0; i < 4; ++i) {
            wheels[i] = w[i];
        }
    }

    void start() {
        std::cout << "Car model " << model << " is starting." << std::endl;
        engine->start();
    }

    void display() {
        std::cout << "Car model: " << model << std::endl;
        std::cout << "Engine type: " << engine->getType() << std::endl;
        std::cout << "Wheel sizes: ";
        for (int i = 0; i < 4; ++i) {
            std::cout << wheels[i]->getSize() << " ";
        }
        std::cout << std::endl;
    }
};
```

### Example Usage

```cpp
int main() {
    Engine engine("V8");
    Wheel* wheels[4];
    for (int i = 0; i < 4; ++i) {
        wheels[i] = new Wheel(18);
    }

    Car car("Mustang", &engine, wheels);
    car.start();
    car.display();

    // Clean up
    for (int i = 0; i < 4; ++i) {
        delete wheels[i];
    }

    return 0;
}
```

### Explanation

1. **Engine Class**: Represents the engine of the car with a type and a method to start the engine.
2. **Wheel Class**: Represents a wheel with a size.
3. **Car Class**: Represents a car that has an engine and four wheels (aggregation).

In the `Car` class, the engine and wheels are passed as parameters to the constructor and are stored as pointers. This indicates that the `Car` class uses these objects but does not own them. The engine and wheels can exist independently of the car.

### Output

```
Car model Mustang is starting.
Engine of type V8 is starting.
Car model: Mustang
Engine type: V8
Wheel sizes: 18 18 18 18 
```

### Summary

Aggregation in C++ is demonstrated by creating a `Car` class that contains an `Engine` and a set of `Wheel` objects. The `Car` class aggregates these objects, meaning it uses them but does not own their lifecycle. This relationship models real-life scenarios where components can exist independently of the object they are part of, such as an engine or wheels existing independently of a car.
