### Types of Destructors with Real-Life Examples

In object-oriented programming, a destructor is a special member function of a class that is executed when an object of that class is destroyed. Destructors are primarily used to release resources allocated to an object during its lifetime. Unlike constructors, there is typically only one destructor per class, and it does not take any parameters nor return any value.

However, in certain advanced contexts, you might encounter terms like "virtual destructors" or "pure virtual destructors." These concepts pertain to how destructors behave in the context of inheritance and polymorphism. Let’s go through these concepts with real-life examples.

### 1. Destructor

A **destructor** is automatically called when an object goes out of scope or is explicitly deleted. It is used to clean up resources that the object may have acquired during its lifetime.

#### Real-Life Example: Hotel Room Cleanup

Imagine a hotel room (object) that needs to be cleaned up (destructor) after a guest checks out (object goes out of scope).

```cpp
#include <iostream>
using namespace std;
class HotelRoom {
public:
    // Constructor
    HotelRoom() {
        cout << "Hotel room is booked." << endl;
    }

    // Destructor
    ~HotelRoom() {
        cout << "Hotel room is cleaned up." << endl;
    }
};

int main() {
    {
        HotelRoom room1; // Hotel room is booked
    } // Hotel room goes out of scope and is cleaned up

    return 0;
}
```

### 2. Virtual Destructor

A **virtual destructor** ensures that the destructor of the derived class is called when an object is deleted through a pointer to the base class. This is crucial for proper resource cleanup in inheritance hierarchies.

#### Real-Life Example: Base and Derived Class (Library and Section)

Imagine a library (base class) with different sections (derived classes) that need specific cleanup when closing.

```cpp
#include <iostream>
using namespace std;
class Library {
public:
    Library() {
        cout << "Library is opened." << endl;
    }

    virtual ~Library() { // Virtual destructor
        cout << "Library is closed." << endl;
    }
};

class Section : public Library {
public:
    Section() {
        cout << "Section is set up." << endl;
    }

    ~Section() {
        cout << "Section is cleaned up." << endl;
    }
};

int main() {
    Library* lib = new Section(); // Opening a specific section of the library
    delete lib; // Cleaning up the specific section and then the library

    return 0;
}
```

### 3. Pure Virtual Destructor

A **pure virtual destructor** is a destructor that is declared in a base class with no implementation, making the class abstract. This forces derived classes to provide an implementation for the destructor.

#### Real-Life Example: Abstract Resource Manager

Imagine an abstract resource manager (base class) that requires specific cleanup methods for different types of resources (derived classes).

```cpp
#include <iostream>
using namespace std;
class ResourceManager {
public:
    virtual ~ResourceManager() = 0; // Pure virtual destructor
};

ResourceManager::~ResourceManager() {
    cout << "Resource Manager base cleanup." << endl;
}

class FileManager : public ResourceManager {
public:
    FileManager() {
        cout << "File Manager is initialized." << endl;
    }

    ~FileManager() {
        cout << "File Manager specific cleanup." << endl;
    }
};

int main() {
    ResourceManager* rm = new FileManager(); // Initialize file manager
    delete rm; // Clean up file manager and resource manager

    return 0;
}
```

### Summary

- **Destructor**: Automatically called to clean up resources when an object goes out of scope or is explicitly deleted. Used for basic cleanup.
  - **Real-Life Example**: Cleaning up a hotel room after checkout.

- **Virtual Destructor**: Ensures proper cleanup in an inheritance hierarchy when an object is deleted through a base class pointer. Crucial for polymorphic behavior.
  - **Real-Life Example**: Cleaning up specific sections of a library before closing the entire library.

- **Pure Virtual Destructor**: Declares an abstract class, enforcing derived classes to implement their own destructors. Useful for abstract base classes that define a common interface.
  - **Real-Life Example**: An abstract resource manager requiring specific cleanup methods for different resource types.

These concepts help manage resources efficiently, ensuring proper cleanup and preventing resource leaks in object-oriented programs.
